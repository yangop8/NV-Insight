## 1. NVIDIA 发布 Nemotron 3 Nano Omni 开放多模态模型

【快讯内容】
NVIDIA 发布 Nemotron 3 Nano Omni，30B 参数混合专家架构（仅激活 3B），统一文本、图像、视频、音频理解，吞吐量较同类开放模型提升 9 倍，可在 DGX Spark、Jetson 等边缘设备上运行，权重、数据集和训练方案完全开放。

【观点】
Nemotron 3 Nano Omni 的 30B-A3B MoE 架构是边缘多模态推理的重要突破。仅 3B 活跃参数意味着该模型可部署在 Jetson Orin/Thor 级边缘平台上实现实时感知-推理-行动闭环，对嵌入式软件而言，这消除了此前需要分别部署视觉、语音、语言多个独立模型的复杂性。在系统软件层面，MoE 架构对推理引擎的专家路由调度、内存管理提出新要求，vLLM/SGLang 等框架需针对稀疏激活模式优化内核。在网络方面，该模型支持多智能体协作（与 Nemotron 3 Super/Ultra 配合），边缘-云端协同推理对低延迟通信链路有直接需求。

> 参考：
> - [NVIDIA Launches Nemotron 3 Nano Omni Model](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/)
> - [NVIDIA Nemotron 3 Nano Omni Powers Multimodal Agent Reasoning](https://developer.nvidia.com/blog/nvidia-nemotron-3-nano-omni-powers-multimodal-agent-reasoning-in-a-single-efficient-open-model/)

## 2. OpenAI GPT-5.5 基于 NVIDIA GB200 NVL72 基础设施驱动 Codex

【快讯内容】
OpenAI 发布 GPT-5.5 并部署在 NVIDIA GB200 NVL72 十万 GPU 集群上，实现每百万 token 成本降低 35 倍、每兆瓦 token 吞吐提升 50 倍。超过一万名 NVIDIA 员工已在使用 GPT-5.5 驱动的 Codex 进行日常工作。

【观点】
GB200 NVL72 机架级系统的 35 倍成本效率和 50 倍能效提升，标志着大规模推理基础设施进入新阶段。从系统软件角度看，十万 GPU 集群的联合调试（joint bring-up）和系统级可靠性基准刷新，对多节点调度、GPU 驱动/固件协同、故障恢复机制提出了极高要求。在网络层面，NVL72 机架间的 NVLink 互联以及跨机架的高带宽通信是达成此性能的关键，Spectrum-X 以太网在此类超大规模部署中的角色日益凸显。

> 参考：
> - [OpenAI's New GPT-5.5 Powers Codex on NVIDIA Infrastructure](https://blogs.nvidia.com/blog/openai-codex-gpt-5-5-ai-agents/)

【快讯内容】

## 4. NVIDIA 与能源行业合作推出弹性供电 AI 工厂架构

【快讯内容】
NVIDIA 联合 Emerald AI 及 AES、Constellation 等能源公司发布弹性供电 AI 工厂方案，基于 Vera Rubin DSX 参考设计和 Emerald AI Conductor 平台，使 AI 工厂能作为弹性电网资产动态调节功耗，同时保持推理性能。Silicon Valley Power 已启动试点。

【观点】
弹性供电架构将计算、供电网络和控制整合为一体，对系统软件的工作负载调度提出了新维度——需在保证推理 SLA 的同时响应电网信号进行动态功率调节。这要求 GPU 驱动和固件层面支持细粒度功耗控制，调度器需具备能效感知能力。从网络角度看，该架构需要实时遥测和控制面通信，与 Spectrum 交换机的网络遥测能力形成互补。

> 参考：
> - [Efficiency at Scale: NVIDIA, Energy Leaders Accelerating Power-Flexible AI Factories](https://blogs.nvidia.com/blog/energy-efficiency-ai-factories-grid/)

## 5. Omniverse 发布模块化库，支持物理 AI 集成至现有应用

【快讯内容】
NVIDIA 发布 Omniverse 模块化库架构，将 RTX 渲染、PhysX 仿真和数据存储管线封装为独立 C API（ovrtx、ovphysx、ovstorage），提供 C++ 和 Python 绑定，支持无头模式优先运行，便于集成至现有工业软件和机器人训练管线。

【观点】
Omniverse 从整体平台向模块化 C API 库转型，对系统软件开发者意义重大。headless-first 设计和独立 C API 降低了集成门槛，使 PhysX 物理仿真和 RTX 渲染可直接嵌入 Isaac Lab 等机器人训练管线，无需完整 Omniverse 运行时。对嵌入式软件而言，这种轻量化集成模式更适合资源受限的边缘仿真场景。

> 参考：
> - [Integrate Physical AI Capabilities into Existing Apps with NVIDIA Omniverse Libraries](https://developer.nvidia.com/blog/integrate-physical-ai-capabilities-into-existing-apps-with-nvidia-omniverse-libraries/)
