## 1. MiniMax M3 登陆 NVIDIA 加速基础设施

【快讯内容】
MiniMax M3 在 NVIDIA Blackwell 等加速平台上线，428B 参数 MoE、原生多模态、支持 100 万 token 上下文。核心创新 MiniMax 稀疏注意力（MSA）以预筛选块替代二次注意力，KV cache 块单次连续读取，较现有稀疏注意力快 4 倍以上，可经 build.nvidia.com 或 Hugging Face 获取。

【观点】
系统软件方向：MSA 把长上下文推理瓶颈从二次注意力转为块级稀疏访问，KV cache 连续内存读取对显存带宽与内核调度友好，是 vLLM/TensorRT/SGLang 类引擎在百万 token 场景的关键优化路径；428B MoE 的多节点部署进一步考验调度与 NVLink/网络互联。嵌入式方向：1M 上下文与多模态对端侧显存仍偏重，主要面向数据中心而非端侧。

> 参考文章：[Deploy Long-Context Reasoning and Agentic Workflows with MiniMax M3 on NVIDIA Accelerated Infrastructure](https://developer.nvidia.com/blog/deploy-long-context-reasoning-and-agentic-workflows-with-minimax-m3-on-nvidia-accelerated-infrastructure)

## 2. ACE Game Agent SDK 与 UE5 端侧 AI 伙伴

【快讯内容】
NVIDIA 在 Unreal Fest 2026 发布 ACE Game Agent SDK（Beta）及 UE5 插件，集成 ASR、小语言模型、TTS，全部低延迟端侧运行于 RTX。轻量开源 C/C++ 框架含 Agent/Chat/RAG 三类 API，可定制，已在 PUBG、Total War: PHARAOH 实测。

【观点】
嵌入式软件方向：典型端侧 AI 部署范式——小模型量化 + RTX 硬件加速实现实时低延迟推理，C/C++ 轻量框架贴近原生集成、内存占用可控，构成端侧 Agent（ASR-SLM-TTS 流水线）工程模板。系统软件方向：Agent API 的自主多步工具调用与 RAG 本地知识库检索，对端侧运行时编排、调度与内存隔离提出要求。网络方向参考价值有限。

> 参考文章：[Build On-Device AI Companions with the NVIDIA ACE Game Agent SDK and Unreal Engine 5 Plugins](https://developer.nvidia.com/blog/build-on-device-ai-companions-with-the-nvidia-ace-game-agent-sdk-and-unreal-engine-5-plugins/)
