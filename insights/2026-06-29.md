## 1. Nemotron 3 Ultra NVFP4 量化检查点：单一检查点跨 Hopper/Blackwell 自适应

【快讯内容】
NVIDIA 用 Model Optimizer 将 550B 的 Nemotron 3 Ultra 量化为 NVFP4，体积 1121GB→352.3GB（3.2x），精度近乎追平 BF16，解码密集负载吞吐较 GLM-5.1 754B FP4 高至 5.9x；单检查点可按硬件自动切换 Blackwell 原生 W4A4 / Hopper W4A16。

【观点】
面向**系统软件**：NVFP4 的"单检查点、运行时按硬件转换权重格式"将量化产物与部署硬件解耦，推理引擎（TensorRT/vLLM 等）无需为 Hopper、Blackwell 各维护一份权重，简化了多代 GPU 混合集群的模型分发与调度。3.2x 显存压缩把 550B 大模型从需多卡才能装下压到单/少卡可驻留，直接降低 KV 缓存外的权重显存压力。对**嵌入式/边缘**方向，W4A16/W4A4 的同源量化思路也为端侧"一次量化、跨设备部署"提供了范式参考。

> 参考文章：[Creating the NVIDIA Nemotron 3 Ultra NVFP4 Checkpoint with NVIDIA Model Optimizer](https://developer.nvidia.com/blog/creating-the-nvidia-nemotron-3-ultra-nvfp4-checkpoint-with-nvidia-model-optimizer/)

## 2. KRAFTON PUBG Ally：纯端侧 ASR+2B SLM+TTS 的实时可协同 AI 队友

【快讯内容】
KRAFTON 基于 NVIDIA ACE 为《PUBG》打造可协同角色 PUBG Ally，整条 ASR+量化 2B Mistral-NeMo-Minitron SLM+TTS 流水线全程端侧运行，模型仅占游戏外剩余显存、8GB 显存 GPU 即可跑；采用 System 1/System 2 架构、KV 缓存优化与智能体工具调用。公测 06-17 至 06-30。

【观点】
面向**嵌入式软件**：这是端侧实时 AI 推理部署的范本——在 8GB 显存预算内、与游戏争抢资源的前提下，靠 2B 量化 SLM + 知识蒸馏把延迟压到实时战斗可用，印证了"模型量化+内存优化"对端侧落地的决定性作用。**系统软件**视角：System 1/System 2 双系统（行为树快反应 vs 语言慢推理）+ 智能体工具调用获取游戏状态，是把 LLM 嵌入实时调度回路、兼顾低延迟与可控性的工程化解法；KV 缓存优化则是端侧推理内核调优的关键抓手。

> 参考文章：[Q&A: How KRAFTON Built PUBG Ally, a Co-Playable Character Powered by NVIDIA ACE](https://developer.nvidia.com/blog/how-krafton-built-pubg-ally-a-co-playable-character-powered-by-nvidia-ace/)
