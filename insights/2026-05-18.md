## 1. Spectrum-X 以太网引入 MRC 多路径可靠连接协议

【快讯内容】
NVIDIA 联合 Microsoft、OpenAI 等发布 MRC（Multipath Reliable Connection）RDMA 传输协议，已作为开放 OCP 规范公开。MRC 可将单条 RDMA 连接分散至多条网络路径，提升吞吐量、负载均衡与可用性，已在 OpenAI、Microsoft、Oracle 的大规模 AI 工厂投产。

【观点】
MRC 是 Spectrum-X 以太网平台的关键协议层升级。对网络方向，它将多路径负载均衡和故障恢复能力内建到 RDMA 传输层，直接提升大规模 GPU 集群训练的网络利用率。该协议由 NVIDIA、AMD、Broadcom、Intel 联合贡献并开放为 OCP 规范，意味着以太网 AI 工厂网络正走向标准化。对系统软件而言，MRC 的遥测驱动拥塞控制和自适应路由需与交换机固件深度协同，Cumulus Linux / NetQ 的拥塞感知调度将更加关键。

> 参考：[Spectrum-X MRC for Gigascale AI](https://blogs.nvidia.com/blog/spectrum-x-ethernet-mrc/)

## 2. NVIDIA Dynamo 1.0 正式发布，支持多节点分布式推理

【快讯内容】
NVIDIA Dynamo 1.0 在 GTC 2026 后进入 GA 阶段，支持分离式 prefill/decode 推理，结合 GB200 NVL72 宽专家并行可实现 7 倍性能提升。已被 ByteDance、CoreWeave、DigitalOcean 等十余家企业部署到生产环境，核心组件 NIXL 被 vLLM、SGLang、TensorRT-LLM 集成。

【观点】
Dynamo 1.0 GA 标志着分离式推理（disaggregated serving）从实验走向工业化。对系统软件方向，其 prefill/decode 阶段独立调度机制重新定义了 GPU 资源分配方式，gang scheduling 和拓扑感知放置对 Kubernetes 调度器提出新要求。NIXL 作为独立通信库被主流推理引擎采纳，正在形成推理数据平面的事实标准。

> 参考：[NVIDIA Dynamo 1.0 Powers Multi-Node Inference](https://developer.nvidia.com/blog/nvidia-dynamo-1-production-ready/)

## 3. Dynamo 全栈优化面向 Agentic 推理场景

【快讯内容】
NVIDIA 发布 Dynamo 针对 Agent 推理的全栈优化方案，包括 agent_hints API 让编排器向推理引擎传递结构化信号、LLM 感知路由与 KV cache 热管理。实测 Agent 工作流可达 85-97% 缓存命中率，4 个 Agent 协作场景聚合命中率达 97.2%。

【观点】
Agent 推理正成为推理引擎优化的新主战场。对系统软件方向，agent_hints API 打破了推理引擎将请求视为匿名 token 流的传统模式，转向语义感知调度。KV cache 跨 worker 路由与热管理直接影响多轮对话的延迟和吞吐，这对推理内核的内存管理和进程间通信设计提出更高要求。

> 参考：[Full-Stack Optimizations for Agentic Inference with Dynamo](https://developer.nvidia.com/blog/full-stack-optimizations-for-agentic-inference-with-nvidia-dynamo/)

## 4. Dynamo 新增流式 Token 与多轮 Agent 工具调用支持

【快讯内容】
NVIDIA Dynamo 新增流式 tool call 派发能力，将推理过程中的 reasoning 段、tool-call 事件和元数据实时流式返回，而非等待整轮完成。解析器、协议和分词器层已抽取为独立可复用 crate（dynamo-protocols、dynamo-parsers、dynamo-tokenizers）。

【观点】
对系统软件方向，流式 tool dispatch 从根本上改变了 Agent 推理的交互模型——从请求-响应转向事件流。独立 crate 化的协议层设计使第三方可在不 fork Dynamo 的前提下构建定制推理服务前端，降低了推理引擎二次开发的门槛。这对 vLLM/SGLang 等开源引擎的 Agent 支持路径有参考价值。

> 参考：[Streaming Tokens and Tools in NVIDIA Dynamo](https://developer.nvidia.com/blog/streaming-tokens-and-tools-multi-turn-agentic-harness-support-in-nvidia-dynamo/)

## 5. NVIDIA Fleet Intelligence 正式可用，面向数据中心 GPU 舰队监控

【快讯内容】
NVIDIA Fleet Intelligence 正式 GA，提供基于 Agent 的 GPU 遥测管理服务，监控功耗、温度、利用率、互连健康等指标。主机端 Agent 以开源方式发布，支持审计。数据流式上传至全托管云服务进行分析。

【观点】
对系统软件方向，Fleet Intelligence 填补了大规模 GPU 集群运维观测的空白。其开源 Agent 设计意味着可与现有 Prometheus/Grafana 监控栈集成，且低开销遥测采集对驱动层和固件的性能影响可控。对网络方向，互连健康监控（NVLink/PCIe/RoCE）有助于快速定位训练任务中的网络瓶颈。

> 参考：[NVIDIA Fleet Intelligence for GPU Fleet Visibility](https://developer.nvidia.com/blog/introducing-nvidia-fleet-intelligence-for-real-time-gpu-fleet-visibility-and-optimization/)

## 6. NVIDIA Model Optimizer 发布训练后量化指南

【快讯内容】
NVIDIA 发布基于 Model Optimizer 的训练后量化（PTQ）技术博客，支持 FP4、FP8、INT8、INT4 等格式及 SmoothQuant、AWQ、SVDQuant 等算法。NVFP4 可将模型内存占用较 FP16 降低 3.5 倍，精度损失控制在 1% 以内。

【观点】
对嵌入式软件方向，NVFP4 量化直接拓展了 Jetson/DGX Spark 等边缘设备可运行的模型规模边界。3.5 倍内存压缩使得百亿参数模型在 128GB 统一内存设备上可行。对系统软件方向，ModelOpt 与 TensorRT 的深度集成意味着量化不再是独立步骤，而是嵌入推理引擎编译管线的一环。

> 参考：[Model Quantization: PTQ Using NVIDIA Model Optimizer](https://developer.nvidia.com/blog/model-quantization-post-training-quantization-using-nvidia-model-optimizer/)

## 7. DGX Spark 软件优化：NVFP4 加速本地推理

【快讯内容】
DGX Spark 新一轮软件更新引入 NVFP4 数据格式支持和投机解码，Qwen-235B 模型在双 DGX Spark 配置下较 FP8 执行性能提升 2.6 倍。新增 NVIDIA Brev 集成和 Hugging Face 教程，简化混合本地-云端 AI 工作流部署。

【观点】
对嵌入式软件方向，DGX Spark 定位为桌面级 AI 工作站，2.6 倍性能提升使 235B 参数模型的本地推理从"能跑"进入"可用"阶段。投机解码（speculative decoding）与 NVFP4 的结合是当前端侧大模型推理加速的关键技术路径。对系统软件方向，ConnectX-7 高速网络支持预示混合推理架构中本地节点与云端的协同将更加紧密。

> 参考：[New Software and Model Optimizations Supercharge DGX Spark](https://developer.nvidia.com/blog/new-software-and-model-optimizations-supercharge-nvidia-dgx-spark/)
