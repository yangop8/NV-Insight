## 1. AI 工厂效率的终极指标：每瓦性能

【快讯内容】
NVIDIA 提出"每瓦性能"是功耗受限时代 AI 工厂的立身之本——固定功耗预算下能产出多少 token 直接决定营收与利润，且该指标无法作弊、只能靠实测挣得。文章强调前沿模型几乎全为 MoE 架构，高效服务的关键在 GPU 域规模，即经超高速 scale-up 互联相连的 GPU 数量，越大越好。

【观点】
面向网络：GPU 域大小取决于 NVLink/NVSwitch scale-up 互联带宽与拓扑，MoE 专家并行的全对全（all-to-all）通信是域内关键路径，直接影响每瓦有效吞吐；面向系统软件：将 token 经济学与功耗预算绑定，推动运行时按性能/瓦做 MoE 专家路由、批调度与功率封顶（power-capping）联合优化。对网络与系统软件方向有直接参考价值。

> 参考文章：[Why Performance per Watt Is the Ultimate Metric for AI Infrastructure Efficiency](https://blogs.nvidia.com/blog/performance-per-watt-ai-infrastructure-efficiency/)

## 2. 用 TAO Agent Skills 一天内完成 Cosmos 3 后训练

【快讯内容】
NVIDIA 将 Cosmos 3 世界基础模型与 TAO agent skills + LoRA 结合，实现视觉语言推理的全自动后训练：agent skills 封装数据处理、基线评估、LoRA 配置与 TAO AutoML 超参寻优，在 Woven 交通安全数据集上一天内把准确率从 54.41% 提到 93.35%。部署经 Cosmos 3 Reasoner NIM 直接把 LoRA 适配器作为 OpenAI 兼容端点提供。

【观点】
面向嵌入式：Cosmos 3 混合 Transformer 分离推理/生成通路，AV 交通安全视觉语言推理与 NIM 端侧/边缘服务化，利于物理 AI 快速领域适配；面向系统软件：TAO AutoML 把后训练流水线技能化，NIM 微服务消除 CUDA 依赖与基础设施复杂度、以标准端点交付 LoRA。对嵌入式与系统软件方向有直接参考价值。

> 参考文章：[Post-Train NVIDIA Cosmos 3 in One Day Using Agent Skills](https://developer.nvidia.com/blog/post-train-nvidia-cosmos-3-in-one-day-using-agent-skills/)

## 4. 用 RL Agent Skills 与 NeMo 运行自研究（Autoresearch）工作流

【快讯内容】
NVIDIA 展示以 NeMo RL、NeMo Gym、Brev、Codex 加一组操作技能（Brev-etiquette 系统卫生、session-memory 状态持久化、autoresearch 实验管理）自动化强化学习研究：Codex（GPT 5.5）端到端完成环境搭建、实验编排与迭代优化，把论文算法（如 OAPL 离策略 RL）落成可运行代码，在自定义视觉语言环境把准确率从 25% 提到 96.9%，全程可复现、单仓库账本化追踪。

## 5. 用引导式生成模型估计极端事件概率（附录登记）

【快讯内容】
NVIDIA 介绍以引导式扩散气候仿真器（如 cBottle）对稀有极端天气做重要性采样：将生成模型引向低概率状态并用 odds-ratio 校正估计其在基线气候分布下的真实概率。该 odds-ratio 诊断需对生成模型求二阶导，量化引导带来的似然偏移并对样本重加权，在热带气旋风险估计中较传统蒙特卡洛显著降低标准误，已实现于 Earth2Studio。

【观点】
属气候科学建模应用。仅在系统软件层面涉及扩散模型推理与二阶自动微分（对生成模型二阶导计算的算子/内存开销）有间接背景意义，与网络、嵌入式软件、系统软件三方向无直接参考价值，仅作净新增登记。

> 参考文章：[Extreme Event Likelihoods with Guided Generative Models](https://developer.nvidia.com/blog/extreme-event-likelihoods-with-guided-generative-models/)
