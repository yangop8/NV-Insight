## 1. Spectrum-X 引入 MRC 多路径可靠连接协议

【快讯内容】
NVIDIA 联合微软、OpenAI 发布 MRC（Multipath Reliable Connection）开放 RDMA 传输协议，单连接可跨多路径分发流量，提升大规模 AI 训练网络的吞吐与可用性。已通过 OCP 开源。

【观点】
MRC 是 Spectrum-X 以太网平面从"够用"迈向"替代 InfiniBand"的关键一步。传统 RoCE 单路径模型在万卡集群中遇到拥塞即全局等待，MRC 的多路径负载均衡和故障绕行能力直接解决了这一痛点。对网络软件栈而言，DOCA SDK 和 Cumulus 路由策略需适配 MRC 的多路径调度逻辑；对系统软件而言，驱动层面 ConnectX 网卡固件需支持 MRC 连接管理，这将推动 RDMA 内核模块的重构。该协议以 OCP 开源方式发布，有助于推动行业标准统一。

> 参考：[NVIDIA Spectrum-X — the Open, AI-Native Ethernet Fabric — Sets the Standard for Gigascale AI, Now With MRC](https://blogs.nvidia.com/blog/spectrum-x-ethernet-mrc/)

## 2. Nemotron 3 Nano Omni 多模态模型发布

【快讯内容】
NVIDIA 发布 Nemotron 3 Nano Omni 开放多模态模型，30B 参数仅 3B 活跃，统一视觉/音频/语言，可在 Jetson、DGX Spark 等边缘设备上部署，推理吞吐较同类模型提升 9 倍。

【观点】
3B 活跃参数的 MoE 架构使该模型成为 Jetson AGX Orin/Thor 平台上多模态 Agent 的强力候选。对嵌入式软件开发者而言，单一模型替代原先视觉+语音+NLP 的多模型流水线，大幅降低边缘部署的集成复杂度和内存占用。系统软件方面，MoE 推理的专家路由调度、KV cache 管理和 NVFP4 量化支持将推动 TensorRT-LLM/vLLM 在边缘场景的优化。

> 参考：[NVIDIA Launches Nemotron 3 Nano Omni Model, Unifying Vision, Audio and Language for AI Agents](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/)

## 3. Fleet Intelligence GPU 舰队监控服务正式发布

【快讯内容】
NVIDIA Fleet Intelligence 正式 GA，提供基于 Agent 的数据中心 GPU 实时遥测监控，覆盖功耗、温度、性能、健康和配置一致性。Agent 已开源，支持 Hopper/Blackwell/Vera Rubin 系统。

【观点】
Fleet Intelligence 的开源 Agent 架构为系统软件团队提供了生产级 GPU 舰队管理的参考实现。其遥测数据涵盖 GPU、NVLink、网络互连健康状态，对网络运维尤其有价值——可实现 NVLink/RoCE 链路故障的早期检测和主动告警。对驱动和固件团队而言，Agent 与 GPU 驱动的交互接口定义了标准化的监控 API 模式，值得参考。

> 参考：[Introducing NVIDIA Fleet Intelligence for Real-Time GPU Fleet Visibility and Optimization](https://developer.nvidia.com/blog/introducing-nvidia-fleet-intelligence-for-real-time-gpu-fleet-visibility-and-optimization/)

## 4. NVIDIA Model Optimizer 发布训练后量化指南

【快讯内容】
NVIDIA 发布 TensorRT Model Optimizer 训练后量化（PTQ）技术详解，支持 SmoothQuant、AWQ、AutoQuantize 等方法及 NVFP4 格式，集成 vLLM/SGLang/TensorRT-LLM。

【观点】
对嵌入式软件而言，NVFP4/INT4/W4A16 等低精度格式在 Jetson 平台上可显著降低内存占用，使 10B 级模型在边缘可用。AutoQuantize 基于梯度的逐层敏感度评分机制降低了量化调优门槛。系统软件方面，Model Optimizer 与 vLLM、SGLang 等推理引擎的深度集成意味着量化模型可直接进入生产推理流水线，减少从研发到部署的工具链断裂。

> 参考：[Model Quantization: Post-Training Quantization Using NVIDIA Model Optimizer](https://developer.nvidia.com/blog/model-quantization-post-training-quantization-using-nvidia-model-optimizer/)

## 5. 车载 AI Agent 从云到车全栈方案

【快讯内容】
NVIDIA 发布从云到车的车载 AI Agent 构建指南，基于 DRIVE AGX Orin 边缘平台，将多模态 Agentic AI 部署到智能座舱，JLR 等车企已采用 DRIVE AI 平台量产。

【观点】
车载场景是嵌入式 AI 最严苛的应用之一——实时性、功耗约束和功能安全（ASIL）缺一不可。从云端训练到 DRIVE AGX Orin 边缘推理的全栈路径，要求嵌入式软件团队在模型压缩、RTOS 实时调度和安全认证三方面协同。对网络而言，云-车数据同步的低延迟通信协议设计和 OTA 更新机制也是关键环节。

> 参考：[How to Build In-Vehicle AI Agents with NVIDIA: From Cloud to Car](https://developer.nvidia.com/blog/how-to-build-in-vehicle-ai-agents-with-nvidia-from-cloud-to-car/)

## 6. Agentic 系统极致协同设计与 Vera Rubin 平台

【快讯内容】
NVIDIA 发布 Agentic 系统极致协同设计白皮书，阐述 Vera Rubin NVL72 平台如何通过七芯片协同将万亿参数 MoE 模型推理成本降至 Blackwell 的 1/10，单用户 400+ tokens/s。

【观点】
极致协同设计的核心在于打破传统芯片边界——Vera CPU 处理 Agent 工具调用和 KV cache 卸载，Rubin GPU 负责 MoE 推理，NVLink 6 提供 3.6TB/s 每 GPU 带宽。对系统软件而言，CPU-GPU 统一执行框架和 KV cache 分层管理是新的优化方向。对网络而言，NVL72 机架内 260TB/s 的 NVLink 互连和机架间 Spectrum-6/ConnectX-9 的配合定义了下一代 AI 工厂网络拓扑基准。

> 参考：[Building for the Rising Complexity of Agentic Systems with Extreme Co-Design](https://developer.nvidia.com/blog/building-for-the-rising-complexity-of-agentic-systems-with-extreme-co-design/)
