## 2. OpenAI GPT-5.5 基于 NVIDIA GB200/GB300 NVL72 运行 Codex

【快讯内容】
GPT-5.5 在 GB200/GB300 NVL72 上实现 50 倍吞吐量/瓦提升和 35 倍成本降低，NVIDIA 内部万人已用 Codex 辅助工程开发。

【观点】
GB300 NVL72 rack-scale 架构依赖 NVLink 和高速以太网实现多芯片互联，50 倍吞吐量/瓦的提升反映了网络带宽与计算的协同优化。TensorRT-LLM 和 NVIDIA Dynamo 在推理引擎层面持续优化内核调度，四个月内 GB200 低延迟性能提升 5 倍，体现系统软件深度调优能力。OpenAI 承诺部署超 10GW NVIDIA 系统，大规模集群的网络拓扑设计将成为关键瓶颈。

> 参考：[OpenAI's New GPT-5.5 Powers Codex on NVIDIA Infrastructure](https://blogs.nvidia.com/blog/openai-codex-gpt-5-5-ai-agents/)

## 5. 利用 AI 物理仿真加速清洁模块化核反应堆设计

## 6. NemoClaw + OpenClaw：构建安全的本地 AI 代理

【快讯内容】
NemoClaw 开源参考栈一键部署 OpenClaw AI 代理，集成 OpenShell 沙箱运行时，支持 DGX Spark 和 RTX PC 本地运行。

【观点】
OpenShell 运行时引入策略化沙箱和进程隔离机制，是运行时安全（sandbox/process isolation）在 AI 代理场景的具体实践，直接关联系统软件方向的安全研究。在 DGX Spark 等本地设备上部署自主代理，对嵌入式侧的资源管理和推理引擎优化（Nemotron 本地推理 + Ollama）提出新需求。

> 参考：[Build a More Secure, Always-On Local AI Agent with OpenClaw and NVIDIA NemoClaw](https://developer.nvidia.com/blog/build-a-secure-always-on-local-ai-agent-with-nvidia-nemoclaw-and-openclaw/)

## 7. Omniverse 模块化库开放 Physical AI 集成能力

【快讯内容】
NVIDIA 将 Omniverse 核心能力拆分为 ovrtx、ovphysx、ovstorage 三个独立 C API 库，支持 headless 模式集成到现有应用。

【观点】
Omniverse 从整体容器栈转向模块化 C API（附 C++/Python 绑定），大幅降低系统软件集成成本。ovphysx（PhysX 仿真）和 ovrtx（RTX 渲染）以 headless-first 设计面向服务端和嵌入式场景，这一架构转变对边缘侧物理 AI 应用的落地具有实质推动作用。底层 C API 的开放也为 GPU 驱动和固件层面的深度优化提供了更多可能。

> 参考：[Integrate Physical AI Capabilities into Existing Apps with NVIDIA Omniverse Libraries](https://developer.nvidia.com/blog/integrate-physical-ai-capabilities-into-existing-apps-with-nvidia-omniverse-libraries/)
