## 1. NVIDIA 发布 Nemotron 3 Nano Omni 多模态开源模型

【快讯内容】
NVIDIA 发布 Nemotron 3 Nano Omni，30B 参数仅激活 3B/token，25GB 显存可运行，支持视觉/音频/图像/文本统一推理，在 6 项多模态基准中取得开源模型最优，效率达同类 9 倍。

【观点】
3B 激活参数量与 25GB 显存需求使该模型可部署于 Jetson AGX Orin/Thor 等边缘设备，为端侧多模态 Agent 提供了实际可行的推理方案。其稀疏 MoE 架构对推理引擎提出新的内核调度要求，vLLM/TensorRT-LLM 等推理框架需优化专家路由和 KV-cache 管理以适配此类混合激活模型。

> 参考文章：
> - [NVIDIA Launches Nemotron 3 Nano Omni Model, Unifying Vision, Audio and Language for up to 9x More Efficient AI Agents](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/)
> - [NVIDIA Nemotron 3 Nano Omni Powers Multimodal Agent Reasoning in a Single Efficient Open Model](https://developer.nvidia.com/blog/nvidia-nemotron-3-nano-omni-powers-multimodal-agent-reasoning-in-a-single-efficient-open-model/)

## 3. NemoClaw 与 OpenClaw：安全沙箱化本地 AI Agent 部署方案

【快讯内容】
NVIDIA 发布 NemoClaw 开源栈，基于 Nemotron 3 Super 模型，通过 OpenShell 沙箱实现本地 AI Agent 的网络/文件系统隔离和实时策略审批，支持 DGX Spark 等设备部署。

【观点】
NemoClaw 采用 OpenShell 进行进程沙箱隔离和网络/文件系统策略管控，是运行时安全（Runtime Security）在 AI Agent 场景的典型实践。其架构对安全关键系统具有参考意义——Agent 自主执行中的权限边界控制、外部访问审批机制，与嵌入式安全关键系统的 Safety-Critical 认证思路相通。

> 参考文章：
> - [Build a More Secure, Always-On Local AI Agent with OpenClaw and NVIDIA NemoClaw](https://developer.nvidia.com/blog/build-a-secure-always-on-local-ai-agent-with-nvidia-nemoclaw-and-openclaw/)

## 7. Omniverse 模块化库：Physical AI 能力以 C API 形式独立提供

【快讯内容】
NVIDIA 将 Omniverse 核心组件（RTX 渲染、PhysX 仿真、数据存储）拆分为独立 headless-first C API 库（ovrtx、ovphysx、ovstorage），支持 C++/Python 绑定，无需采用完整 Omniverse 容器栈即可集成。

【观点】
模块化 C API 架构降低了 Physical AI 仿真能力的集成门槛，对嵌入式软件开发者尤为关键——可以在既有工业软件和机器人训练管线中按需引入 PhysX 仿真或 RTX 渲染，而无需整体架构重写。headless-first 设计也适合无 GUI 的边缘/服务器环境部署。

> 参考文章：
> - [Integrate Physical AI Capabilities into Existing Apps with NVIDIA Omniverse Libraries](https://developer.nvidia.com/blog/integrate-physical-ai-capabilities-into-existing-apps-with-nvidia-omniverse-libraries/)
