## 1. DiffusionGemma：扩散去噪并行生成文本，突破逐 token 解码瓶颈

【快讯内容】
NVIDIA 联合 Google DeepMind 发布 DiffusionGemma，以扩散去噪并行生成文本 token 而非逐个自回归解码，大幅提升吞吐；可在 H100、DGX Spark/Station、RTX/RTX PRO 上运行，支持 BF16 与 NVFP4，经 Hugging Face、NIM、NeMo AutoModel 部署。

【观点】
- **系统软件**：并行扩散解码改变了推理引擎的核心调度假设——从 KV-cache 顺序生成转向多 token 并行去噪迭代，对 batching、内存复用与算子内核（NVFP4 低精度 matmul）提出全新优化空间，是 vLLM/TensorRT 类引擎值得跟踪的范式变化。
- **嵌入式软件**：支持 DGX Spark 与 RTX PC 端侧部署且提供 NVFP4 量化格式，高并发低延迟特性对端侧实时 copilot/agent 的内存与响应优化有直接参考价值。

> 参考文章：[Run DiffusionGemma on NVIDIA for Developer-Ready, High-Throughput Text Generation](https://developer.nvidia.com/blog/run-diffusiongemma-on-nvidia-for-developer-ready-high-throughput-text-generation/)

## 2. DGX Spark 企业级可管理性：AI 基础设施的全生命周期运维框架

【快讯内容】
NVIDIA 为 DGX Spark / GB10 推出 Enterprise Manageability，提供从首次配置到退役的完整运维框架：采用无 agent 的 SSH 执行模型与标准化 JSON 输出，直接对接 CMDB、SIEM 与监控管道，并支持全气隙（air-gapped）离线部署，解决"已知良好状态"的运维复杂度。

【观点】
- **系统软件**：无 agent SSH + 标准化 JSON 输出的设计，把 AI 节点纳入既有 IT 运维栈（可配置、可观测、可管理），体现系统级运维自动化与 known-good 状态管理思路，对多节点 GPU 集群的生命周期编排有借鉴意义。
- **网络**：气隙/断网环境的安全可管理性，与零信任、运行时隔离方向契合，为受限网络下的 AI 基础设施安全运维提供参考。

> 参考文章：[Delivering Lifecycle Control for AI Infrastructure at Scale with NVIDIA DGX Spark Enterprise Manageability](https://developer.nvidia.com/blog/delivering-lifecycle-control-for-ai-infrastructure-at-scale-with-nvidia-dgx-spark-enterprise-manageability)
