## 快讯一：Spectrum-X 引入 MRC 多路径可靠连接协议

【快讯内容】
NVIDIA 联合微软、OpenAI 推出 MRC（Multipath Reliable Connection）RDMA 传输协议，支持单条 RDMA 连接跨多条网络路径进行流量分发与负载均衡。该规范已通过 OCP 开放发布，AMD、Broadcom、Intel 等参与协作。OpenAI 已用 MRC 在 Spectrum-X 上训练 ChatGPT 前沿模型。

【观点】
MRC 是 Spectrum-X 以太网从"AI 优化"走向"AI 原生"的关键一步。对网络方向而言，MRC 将 RDMA 多路径负载均衡从私有方案升级为开放标准，对 RoCE 大规模部署的拥塞控制和故障恢复能力有实质性提升。对系统软件而言，MRC 可与 ConnectX SuperNIC 的自适应路由协同，降低 GPU 集群通信的尾延迟，直接影响训练和推理吞吐。这是以太网在 AI 工厂场景中对 InfiniBand 的一次正面竞争性回应。

> 参考文章：
> - [NVIDIA Spectrum-X — the Open, AI-Native Ethernet Fabric — Sets the Standard for Gigascale AI, Now With MRC](https://blogs.nvidia.com/blog/spectrum-x-ethernet-mrc/)

## 快讯二：NVIDIA 与 Google Cloud 联手扩展 Vera Rubin AI 工厂

【快讯内容】
NVIDIA 与 Google Cloud 在 Cloud Next '26 上宣布深化合作：Google Cloud AI Hypercomputer 将支持 Vera Rubin 平台 A5X 实例，最大可扩展至近百万颗 Rubin GPU；同时推出基于 Gemini 企业代理平台的 NVIDIA Nemotron 3 Super 模型，用于 agentic 工作流。

【观点】
百万 GPU 级别的 Vera Rubin 部署对网络架构提出极端要求——Spectrum-X Photonics 共封装光学交换机和 ConnectX-9 SuperNIC 成为支撑 scale-out/scale-across 的核心组件。对系统软件而言，NeMo + NIM 的 agentic 推理管线需要与 Rubin 平台的多芯片协同调度深度集成。嵌入式方向可关注 Nemotron 模型未来在边缘侧的精简部署路径。

> 参考文章：
> - [NVIDIA and Google Cloud Collaborate to Advance Agentic and Physical AI](https://blogs.nvidia.com/blog/google-cloud-agentic-physical-ai-factories/)

## 快讯三：Agentic 系统极端协同设计与 Vera Rubin 推理扩展

【快讯内容】
NVIDIA 开发者博客发布两篇系列文章：一是分析 agentic AI 的非确定性 token 消费模式对推理经济性的挑战，提出 prompt 缓存、上下文压缩和 CMX 硬件加速方案；二是阐述 Vera Rubin NVL72 + Groq 3 LPX 组合如何同时满足 agentic 推理的高吞吐与低延迟需求。

【观点】
这两篇文章对系统软件方向价值极高。Agentic 推理的工具调用链和多层 agent 架构会产生数十万 token 的上下文窗口，CMX 硬件与 KV-cache 优化直接影响推理引擎（如 vLLM、TensorRT-LLM）的内核调度策略。对网络方向，NVL72 的 NVLink 6 scale-up 与 Spectrum-X scale-out 的流量分配比例将随 agentic 工作负载特征动态变化，需要网络遥测系统实时适配。

## 快讯四：车载 AI Agent 从云端到座舱的全栈部署

【快讯内容】
NVIDIA 发布从云到车的 AI Agent 构建指南，阐述汽车座舱如何从规则式界面转向 agentic 多模态 AI 系统。方案基于 DRIVE AGX Thor 平台，利用 TensorRT Edge-LLM 在嵌入式端实现 LLM/VLM 实时推理，支持 EAGLE-3 推测解码和 NVFP4 量化。

【观点】
嵌入式软件方向的重点参考。TensorRT Edge-LLM 的轻量级 C++ 框架与 JetPack/DriveOS 深度集成，代表了 NVIDIA 将数据中心级推理能力下沉到边缘端的技术路径。NVFP4 量化与推测解码的组合可显著降低嵌入式平台的内存与算力需求。系统软件方向可关注 Edge-LLM 运行时的进程隔离与安全关键认证方案——车规级 agentic AI 对实时调度和功能安全提出了更高要求。

## 快讯五：Token 工厂经济学——性能/功耗比与电信 AI 工厂

【快讯内容】
NVIDIA 发布两篇关于 AI 工厂经济学的文章：一是提出"每兆瓦 token 吞吐量"作为 AI 工厂核心度量指标，分析 Hopper→Blackwell→Vera Rubin 的性能/功耗曲线；二是为电信运营商描绘从 GPU 按小时计费到 Token-as-a-Service 的转型路径。

【观点】
对网络方向，Token 工厂的经济模型直接驱动网络架构选型——高密度推理集群对 Spectrum-4/SN5600 的 east-west 带宽提出更高要求，而 Spectrum-X Photonics 的能效优势在功耗受限场景下尤为关键。对系统软件，推理引擎的每瓦 token 优化需要从 GPU 内核、内存管理到网络传输的全栈协调。电信 AI 工厂的 token 计量模式可能催生新的 DOCA/BlueField 级别的 DPU 侧计量与安全方案。

## 快讯六：GTC Taipei / COMPUTEX 2026 预览

【快讯内容】
NVIDIA GTC Taipei 将于 6 月 1 日在台北开幕，Jensen Huang 发表主题演讲。Vera Rubin NVL72 获 COMPUTEX 金奖——集成 36 颗 Vera CPU、72 颗 Rubin GPU，通过 ConnectX-9 SuperNIC 和 Spectrum-X Ethernet Photonics 共封装光学交换机实现 scale-out 与 scale-across 网络互联。

【观点】
GTC Taipei 是下半年 AI 基础设施路线图的风向标。对网络方向，Spectrum-X Ethernet Photonics 和新公布的 Spectrum-XGS（scale-across 以太网）的量产时间线将在此次大会上进一步明确，值得重点关注。对系统软件和嵌入式方向，Rubin 平台的六芯片协同设计（Vera CPU + Rubin GPU + NVLink 6 + ConnectX-9 + BlueField-4 + Spectrum-6）意味着驱动和固件层面的复杂度大幅增加，多芯片调度策略将是关键技术挑战。
