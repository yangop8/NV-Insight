## 1. Vera CPU 首批交付顶级 AI 实验室

【快讯内容】
NVIDIA 宣布首批 Vera CPU 交付 Anthropic、OpenAI、SpaceXAI 及 Oracle Cloud。Vera 集成 88 个定制 Olympus Arm 核心、2270 亿晶体管、1.2TB/s 内存带宽，专为 AI 智能体工作负载设计。

【观点】
Vera 是 Vera Rubin 平台核心，与 Spectrum-6 交换机、BlueField-4 DPU、ConnectX-9 SuperNIC 协同设计。其强调内存带宽和低延迟互连的架构直接服务于智能体场景的向量数据库访问和 GPU 间数据搬运，将显著影响数据中心网络拓扑设计和系统软件多芯片调度策略。

> 参考：[Vera Arrives: NVIDIA's First CPU Built for Agents Lands at Top AI Labs](https://blogs.nvidia.com/blog/vera-cpu-delivery/)（2026-05-18）

## 2. Nemotron 3 Nano Omni：统一视觉、语音、语言的多模态模型

【快讯内容】
NVIDIA 发布 Nemotron 3 Nano Omni 开源多模态模型，采用 30B-A3B 混合 MoE 架构（Mamba-Transformer），每 token 仅激活 3B 参数，25GB 内存即可运行，吞吐量达同类开源模型 9 倍。

【观点】
该模型支持 Jetson Thor 和 AGX Orin 64GB 部署（Ollama Q4_K_M GGUF），可通过 TensorRT Edge-LLM 加速。混合 Mamba-Transformer 架构和极致稀疏激活设计是嵌入式端侧推理的重要进展——3B 激活参数的多模态能力使机器人、自动驾驶等边缘场景具备实时视觉+语音理解成为可能。

> 参考：[NVIDIA Launches Nemotron 3 Nano Omni Model](https://blogs.nvidia.com/blog/nemotron-3-nano-omni-multimodal-ai-agents/)（2026-05-19）
> 参考：[NVIDIA Nemotron 3 Nano Omni Powers Multimodal Agent Reasoning](https://developer.nvidia.com/blog/nvidia-nemotron-3-nano-omni-powers-multimodal-agent-reasoning-in-a-single-efficient-open-model/)（2026-04-28）

## 3. 面向 Agentic 系统的极致软硬件协同设计

【快讯内容】
NVIDIA 技术博客阐述 AI 智能体系统对极致协同设计的需求：智能体动态调用工具、生成子智能体、管理上下文窗口，推动 token 消耗和延迟需求进入极端区间，需要 Vera Rubin 全栈平台支撑。

【观点】
文章揭示的智能体工作负载特征——高 token 消耗、长上下文、超低延迟——对推理引擎内核优化（vLLM/TensorRT-LLM）和多芯片/多节点调度提出全新要求。软件与芯片并行设计的理念是理解下一代 AI 基础设施系统软件栈演进方向的关键参考。

> 参考：[Building for the Rising Complexity of Agentic Systems with Extreme Co-Design](https://developer.nvidia.com/blog/building-for-the-rising-complexity-of-agentic-systems-with-extreme-co-design/)（2026-05-05）

## 4. 从云到车：构建车载 AI 智能体

【快讯内容】
NVIDIA 技术博客发布车载 AI 智能体构建指南，基于 NeMo 平台覆盖云端训练到车载部署全流程，支持 LLM、多模态模型、RAG 和智能体工作流在 NVIDIA DRIVE 平台推理运行。

## 5. 模型量化：NVIDIA Model Optimizer 训后量化实战

【快讯内容】
NVIDIA 技术博客发布 Model Optimizer（ModelOpt）训后量化指南，演示将 CLIP 模型量化为 FP8 格式的完整流程，涵盖校准、验证和检查点管理，降低 VRAM 占用并提升推理性能。

【观点】
训后量化是嵌入式和边缘部署的关键使能技术。ModelOpt 统一了量化、剪枝、蒸馏、投机解码等优化手段，下游对接 TensorRT-LLM 和 vLLM 等推理引擎，构成从模型压缩到部署加速的完整工具链，对 Jetson 等内存受限平台尤为重要。

> 参考：[Model Quantization: Post-Training Quantization Using NVIDIA Model Optimizer](https://developer.nvidia.com/blog/model-quantization-post-training-quantization-using-nvidia-model-optimizer/)（2026-05-07）

## 6. Hermes 自改进 AI 智能体登陆 RTX PC 和 DGX Spark

【快讯内容】
开源 Hermes Agent 上线三个月 GitHub 星标超 14 万，成为 OpenRouter 最受欢迎智能体。支持在 NVIDIA RTX PC 和 DGX Spark 本地运行自改进智能体工作流。

【观点】
Hermes 在消费级 RTX 和入门级 DGX Spark 上运行自改进智能体，模糊了云端与边缘的推理边界。其本地部署模式对 GPU 驱动/运行时栈和进程隔离提出要求，值得系统软件方向关注。
