## 1. 在 GB200 NVL72 上以 GPU 加速 Presto 跑低延迟分析型负载

【快讯内容】
NVIDIA 展示 GPU 加速 Presto（基于 cuDF 与 Velox 执行引擎）在 DGX B200 与 GB200 NVL72 上运行 TPC-H 分析型查询，较多节点 CPU 集群延迟最低降至 1/8，集群级优化再提速最高 64%。关键在于 cuDF 做查询执行、NVLink 做 GPU 间高速通信、GPUDirect Storage 从 IBM Storage Scale 直取数据入显存以减少 CPU 参与与 NUMA 惩罚。NVL72 测试配 18 节点（每节点 2 Grace+4 B200+4 张 ConnectX-7 400Gbps），已随 GTC 2026 IBM 合作集成进 watsonx.data。

【观点】
面向系统软件：这是把 GPU 从训练/推理扩展到分析型 SQL 查询引擎的系统级案例——cuDF 算子化查询执行、按 GPU 显存与 L2 驻留组织工作集、以 GPUDirect Storage 绕开 CPU 拷贝路径消除 NUMA 惩罚，对数据处理内核优化与显存-存储数据通路调度有直接参考价值；面向网络：NVLink 域内 GPU 间通信与 ConnectX-7 400Gbps + GPUDirect Storage 构成的存储直取数据平面，是分析型负载下 scale-up 互联与存储网络协同的典型拓扑，对低延迟数据搬运与关键路径带宽规划有直接参考价值。嵌入式方向无直接参考价值。

> 参考文章：[Running Low-Latency Analytical Workloads with GPU-Accelerated Presto on NVIDIA GB200 NVL72](https://developer.nvidia.com/blog/running-low-latency-analytical-workloads-with-gpu-accelerated-presto-on-nvidia-gb200-nvl72)
