## 快讯

### 一、Nemotron 3 Nano Omni：边缘多模态推理的新基线

【快讯内容】
NVIDIA 发布 Nemotron 3 Nano Omni 开源多模态模型（4月28日），采用 30B-A3B 混合专家（MoE）架构，统一视觉、语音和语言能力。推理时仅激活 30 亿参数，吞吐量较同类开源模型提升 9 倍，可部署于 Jetson、DGX Spark 等边缘设备。

【观点】
该模型对嵌入式软件方向意义重大：MoE 架构使 30B 参数模型在 Jetson 平台上可用，为机器人和自动驾驶提供了本地多模态推理能力。30 亿活跃参数的轻量设计与 NIM 微服务部署方式降低了端侧 AI 集成门槛，推动端侧智能从单模态向全模态演进。Jetson AI Lab 已提供优化教程和基准测试。

### 二、GPT-5.5 在 NVIDIA GB200 NVL72 上的规模化部署

【快讯内容】
OpenAI 发布 GPT-5.5 并在 NVIDIA GB200 NVL72 机架系统上完成部署（4月23日），双方完成首个 10 万 GPU 集群联合调试。GB200 NVL72 实现每百万 token 成本降低 35 倍、每兆瓦吞吐量提升 50 倍。万余名 NVIDIA 员工已在使用 GPT-5.5 驱动的 Codex。

【观点】
GB200 NVL72 十万 GPU 集群的成功调试标志着 Blackwell 架构在超大规模推理场景的成熟。35 倍成本降低和 50 倍能效提升对系统软件方向影响深远——多节点调度、显存管理和功耗优化将成为充分释放硬件潜力的关键。这也验证了 NVIDIA 推理基础设施从训练向推理工厂转型的战略方向。

### 三、NemoClaw：AI 代理的沙箱安全运行时

【快讯内容】
NVIDIA 发布 NemoClaw 开源栈（4月17日），基于 OpenClaw 和 OpenShell 运行时，实现本地 AI 代理的沙箱化安全部署。支持网络/文件系统隔离、实时策略审批和全本地推理，确保数据不出设备，可通过 Telegram 远程访问。

【观点】
NemoClaw 的沙箱隔离机制直接对应系统软件中的运行时安全需求。OpenShell 的网络隔离和文件系统隔离策略为 AI 代理安全运行提供了参考架构，与 NVIDIA 在零信任架构方面的探索一脉相承。对需要安全关键认证的嵌入式 AI 部署场景（如工业机器人、自动驾驶）也有借鉴价值。

### 四、Omniverse 模块化库：headless-first 的物理 AI 集成

【快讯内容】
NVIDIA 将 Omniverse 核心组件拆分为三个独立模块库（4月8日）：ovrtx（RTX 渲染）、ovphysx（GPU 物理仿真）、ovstorage（数据管道），提供 headless-first C API 及 C++/Python 绑定，可无 UI 集成至工业和机器人软件栈。

【观点】
模块化 C API 的设计与嵌入式系统和高性能系统软件的集成需求高度契合。headless-first 理念使其适合无头服务器和边缘节点部署；ovstorage 的 Kubernetes 微服务架构和 MCP 服务器支持为 AI 工厂数字孪生提供了可扩展基础。Isaac Lab 3.0 Beta 已基于此架构。

### 五、GeForce NOW 更新

【快讯内容】
GeForce NOW 新增 Xbox Game Pass/Ubisoft+ 订阅标签功能（4月23日）；5月新增 16 款云游戏（含《Forza Horizon 6》），RTX 5080 算力扩展至几乎全部目录。

【观点】
没有面向网络、嵌入式软件、系统软件的参考价值。
