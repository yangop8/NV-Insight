## 快讯

### 1. Dynamo Snapshot：GPU 推理进程检查点/恢复，K8s 冷启动降至秒级

【快讯内容】
NVIDIA 发布 Dynamo Snapshot，用 CRIU 与 cuda-checkpoint 对 GPU 推理进程做检查点/恢复，结合 KV 缓存解映射、并行 memfd 恢复与 GPUDirect Storage，将 Kubernetes 冷启动从数分钟降至秒级，gpt-oss-120b 启动提速最高 21 倍，Qwen3-0.6B 冷启动 <5 秒。

【观点】
**系统软件方向**：这是推理服务运行时层的关键工程进展。它用 CUDA 驱动的检查点能力把 GPU 设备状态转储到 CPU 内存，再用 CRIU 序列化整个进程树——本质是把"进程/设备状态快照"从传统 CPU 应用引入到 GPU 推理场景。其中 GPU Memory Service（GMS）将大模型权重与进程状态解耦、并发恢复的设计，对推理引擎（vLLM/SGLang/TensorRT-LLM）的弹性扩缩容和故障恢复有直接借鉴价值；KV 缓存解映射以缩减快照体积的思路，也与分层 KV 缓存管理一脉相承。

**网络方向**：恢复路径依赖 GPUDirect Storage 等高带宽通道，将权重/状态从网络存储并发拉回 GPU，对存储网络的 RDMA 路径和带宽规划提出要求；秒级冷启动也使"按需调度 GPU 推理实例"在网络编排层更具可行性。

**嵌入式方向**：检查点/恢复机制本身对边缘端"快速唤醒/故障切换"的推理部署有概念参考，但本文聚焦数据中心 K8s 场景，对嵌入式平台无直接落地内容。

> 参考文章：[NVIDIA Dynamo Snapshot: Fast Startup for Inference Workloads on Kubernetes](https://developer.nvidia.com/blog/nvidia-dynamo-snapshot-fast-startup-for-inference-workloads-on-kubernetes/)
