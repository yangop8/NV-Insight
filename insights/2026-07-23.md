## 1. 深入 Rubin GPU 架构：为智能体 AI 时代而生

【快讯内容】
Rubin GPU 由两颗光罩极限计算 die 经 NV-HBI 高速互联封装为单芯片，含 3360 亿晶体管、224 个 SM、896 个 Tensor Core，第三代 Transformer Engine 跨数值格式自适应精度。增强 TMA、内联描述符更新、激活稀疏、自适应压缩、细粒度依赖内核触发等优化 MoE 扩展、长上下文注意力并压低内核切换时延，最大化 tokens/秒与 tokens/瓦。机架级 Vera Rubin NVL72 集成液冷、DSX MaxLPS 功率平抑、无缆 MGX 架构与热插拔 NVLink 交换托盘。

【观点】
面向系统软件：细粒度依赖内核触发与增强 TMA 直指推理引擎的内核切换开销与数据搬运瓶颈，激活稀疏/自适应压缩与第三代 Transformer Engine 的跨格式精度自适应，对 MoE 调度、长上下文 KV 管理与内核编排有直接参考价值。面向网络：同功耗封套内多 40% GPU 依赖机架级 scale-up 域（NVLink 交换托盘热插拔）与功率平抑协同，对 scale-up 互联带宽/供电边界的联合规划有直接参考价值。嵌入式方向无直接参考价值。

## 2. Vera CPU 的 Olympus 核：为智能体 AI 打造的极致单线程性能

【快讯内容】
Vera CPU 搭载 88 个自研 Olympus 核，全 Arm 兼容，单核 IPC 较 Grace 高至 50%，靠宽前端、先进分支预测、深乱序调度与专用内存预取支撑分支密集/内存敏感的智能体代码。满 socket 负载下单核性能 1.8x、峰值加载时延低 40%、单核内存带宽 3x 且功耗不足传统 x86 一半。单片 die 配相邻 dielet、每核 14 GB/s 统一内存带宽、LPDDR5X SOCAMM 模组，机架级较 x86 提供 4x 沙箱密度与 2x 每瓦性能。

【观点】
面向系统软件：把 CPU 侧瓶颈定义为"满载单线程性能"而非核数堆叠，对推理引擎中调度、KV-cache 上下文重建、分支密集控制路径的停顿消除有直接参考价值；4x 沙箱密度与统一/确定性内存带宽对运行时隔离（沙箱、进程隔离）与多租户调度是关键杠杆。面向网络：单核内存带宽 3x 与低加载时延支撑 CPU-GPU-NVLink 关键路径尾时延收敛。面向嵌入式：Arm 兼容 + LPDDR5X + 每瓦性能路线与端侧低功耗推理选型逻辑同源，有间接参考价值。

> 参考文章：[NVIDIA Vera CPU: Olympus Cores Built for Maximum Single-Thread Performance in Agentic AI](https://developer.nvidia.com/blog/inside-nvidia-vera-cpu-olympus-cores-built-for-maximum-single-threaded-performance-in-agentic-ai/)

## 3. 在 GB300 NVL72 上创 MoE 预训练世界纪录

【快讯内容】
GB300 NVL72 预训练 DeepSeek-V3 671B 创单 GPU 1648 TFLOPs 世界纪录：第五代 NVLink 提供每 GPU 1.8 TB/s、130 TB/s 无阻塞 all-to-all 带宽做机架内内存语义低时延通信，ConnectX-8 SuperNIC 配 Quantum-X800 InfiniBand 或 Spectrum-X 以太网维持跨机架可预测性能。Megatron Core、TorchTitan、JAX 等软件较前代提速 3x–10x，从 256 卡扩到 1024 卡时单 GPU 吞吐仍保持在 97% 以上。

【观点】
面向网络：这是 MoE 专家并行 all-to-all 通信瓶颈的教科书级实证——scale-up（NVLink 130 TB/s 无阻塞）与 scale-out（ConnectX-8 + Quantum-X800/Spectrum-X）紧耦合决定 97% 强扩展效率，对专家路由通信关键路径与跨机架拓扑规划有直接参考价值。面向系统软件：Megatron Core/TorchTitan/JAX 的 3x–10x 提速表明性能主要来自并行策略与通信-计算重叠的工程化，对训练框架调度、显存与集合通信协同优化有直接参考价值。嵌入式方向无直接参考价值。
