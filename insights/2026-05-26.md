## 3. Vera Rubin 平台解决 Agentic AI 推理扩展难题

【快讯内容】
NVIDIA 技术博客详解 Vera Rubin NVL72 如何应对 Agentic 推理的规模化挑战：Agent 工作流产生非确定性轨迹，单次会话可能触发数百次推理请求并累积端到端延迟，需在万亿参数 MoE 模型上实现持续低延迟、高吞吐生成。

【观点】
Agentic 推理的随机轨迹特性对系统软件的调度策略提出全新要求——传统的批处理优化不再适用，需要细粒度的请求级调度和动态资源分配。推理引擎（vLLM、SGLang）需针对长上下文窗口和 MoE 稀疏激活模式进行内核级优化。网络层面，NVL72 rack-scale 内部的 NVLink 互联和跨机架的 Spectrum-X 以太网需协同处理爆发式的 token 流量。

> 参考：[How the NVIDIA Vera Rubin Platform is Solving Agentic AI's Scale-Up Problem](https://developer.nvidia.com/blog/how-the-nvidia-vera-rubin-platform-is-solving-agentic-ais-scale-up-problem/)

## 4. 车载 AI Agent 从云端到车端的全栈部署

## 6. NVIDIA Dynamo 支持多轮 Agentic 推理的流式 Token 服务

【快讯内容】
NVIDIA Dynamo 推理服务框架新增多轮 Agentic Harness 支持，将工具调用作为类型化分发事件实时流式处理，取代传统的完整回合缓冲模式，提升 Agent 工作流的响应速度。

【观点】
Dynamo 的流式工具分发机制是推理引擎架构的重要演进——将 Agent 的工具调用从同步阻塞转为异步流式处理，对系统软件的运行时调度和进程间通信提出更高要求。这一优化直接影响多 Agent 编排的端到端延迟，是 vLLM/SGLang 等推理引擎需要跟进的方向。

> 参考：[Streaming Tokens and Tools: Multi-Turn Agentic Harness Support in NVIDIA Dynamo](https://developer.nvidia.com/blog/streaming-tokens-and-tools-multi-turn-agentic-harness-support-in-nvidia-dynamo/)
